## 三、关键兼容性风险与处理方案

### 3.1 ✅ Flex Gap（CSS）— 已自动降级

- **风险**：Chrome 84 才支持 flex `gap`，Chrome 80 不支持
- **影响范围**：项目中共 409 处 flex gap 使用
- **处理方案**：`vite.config.js` 中 `flexGapPolyfillPlugin` PostCSS 插件自动生成 `@supports not (gap: 1px)` 回退规则，使用 `> * + *` 的 margin 实现等价布局
- **验证**：构建产物 CSS 中已确认包含 `@supports not (gap: 1px)` 回退规则
- **状态**：✅ 已处理

### 3.2 ✅ Clipboard API — 已降级

- **风险**：Chrome 80 WebView 不支持 `navigator.clipboard`
- **处理方案**：所有复制操作均加 `window.isSecureContext` 判断，HTTP 环境回退到 `document.execCommand('copy')`
- **验证**：设备报告中 `document.execCommand("copy")` 成功
- **状态**：✅ 已处理

### 3.3 ✅ backdrop-filter — 已加 -webkit- 前缀

- **风险**：Chrome 80 需要 `-webkit-backdrop-filter` 前缀
- **处理方案**：所有毛玻璃效果均同时写入 `backdrop-filter` 与 `-webkit-backdrop-filter`
- **状态**：✅ 已处理

### 3.4 ✅ user-select — 已自动加前缀

- **风险**：Chrome 80 需要 `-webkit-user-select` 前缀
- **处理方案**：`webkitPrefixPlugin` PostCSS 插件自动补全
- **状态**：✅ 已处理

### 3.5 ✅ ES 语法 — 已限制为 Chrome 80 可用

- **风险**：Chrome 80 不支持部分 ES2020+ 特性
- **处理方案**：
  - 构建目标 `target: 'chrome80'`
  - 源码遵循 ES5 风格（`var`、`function`，无箭头函数、无可选链）
- **状态**：✅ 已处理

### 3.6 ⚠️ 不支持的 API（业务无依赖）

以下 API 在 Chrome 80 WebView 不可用，但项目未使用，无影响：

- Service Worker / Cache API（项目用 localStorage + apiCache）
- Notification API（项目使用应用内提示）
- DeviceOrientation / DeviceMotion（项目未使用）
- OffscreenCanvas（项目使用主线程 Canvas）
- `structuredClone`（项目使用 JSON.parse/stringify）
- `crypto.randomUUID`（项目使用自实现 UUID）
- MediaDevices（getUserMedia 不可用，项目未使用摄像头/麦克风）

### 3.7 ⚠️ 部分支持的 API（已规避）

| API | 状态 | 规避方案 |
|-----|------|----------|
| `document.execCommand("paste")` | 受限 | 不依赖程序化粘贴，仅用用户交互 |
| `getCurrentPosition()` | 需授权 | 天气页使用固定城市（庐江），不依赖定位 |
| `navigator.getBattery()` | 需交互 | 未使用电池信息 |
| `clipboardData` | 仅事件内 | 仅在 copy 事件中使用 |
