# 🏫 ClassNet 社区系统需求文档

## 📋 系统概述

社区系统是为了提升班级内同学关系而设计的综合性社交平台。系统包含**论坛**、**排行榜**以及**我的**三个核心界面，旨在促进同学之间的交流互动、分享生活点滴。

> **使用设备**：横屏平板\
> **技术要求**：禁用浏览器原生模态框，使用自定义模态框；禁用 ES2020+ 语法（可选链、空值合并等）

***

## 🎯 功能模块

### 一、论坛系统 💬

#### 1.1 发帖功能

- **内容支持**：文字 + Emoji 表情
- **可见性设置**：
  - 默认分组：男 / 女
  - 可查看分组：多选设置
  - 不可查看分组：多选设置
  - 支持用户自定义分组
- **匿名发送**：可选择是否匿名发布
- **权限控制**：
  - 发送者可删除自己的帖子
  - 管理员可删除所有帖子

#### 1.2 互动功能

- **点赞** ❤️：点击即可点赞，显示点赞总数
- **评论** 💭：支持回复评论，形成讨论楼
- **转发** 🔄：转发至聊天系统（需与聊天系统联动）
- **查看用户信息**：点击发送者头像可查看详细信息

#### 1.3 帖子管理

- **删除权限**：
  - 普通用户：仅可删除自己发布的帖子
  - 管理员：可删除任意帖子
- **举报功能**：用户可举报违规内容

***

### 二、排行榜系统 🏆

#### 2.1 美食榜 🍜

**功能描述**：用户分享推荐的美食窗口，按点赞数排行

**发布内容**：

- **菜名**（必填）：例如"牛肉拌面"
- **食堂选择**（下拉框）：
  - 黄山大厦
  - 优芙德
  - 其他（可手动输入）
- **具体窗口**（必填）：例如"优芙德 2 楼 1/2 号窗口"
- **推荐理由**（选填）：口感、特色等描述
- **图片上传** 不支持

**排行规则**：

- 按点赞数量从高到低排序
- 点赞数相同则按发布时间排序

#### 2.2 热事榜 🔥

**功能描述**：分享校园热点事件，用户可自主上传

**发布内容**：

- **事件标题**（必填）
- **事件详情**（必填）：文字描述 + Emoji
- **发生地点**（选填）
- **发生时间**（选填）
- **图片/视频**（选填）

**互动功能**：

- 点赞：按点赞数从高到低排行
- 评论：讨论事件详情
- 转发：分享至聊天系统

***

### 三、个人中心 👤

#### 3.1 我的帖子

查看自己发布的所有内容：

- 论坛帖子
- 美食推荐
- 热事分享

支持操作：

- 删除帖子
- 编辑可见性设置
- 查看点赞/评论数

#### 3.2 个人信息设置

**可编辑信息**：

- 微信号
- QQ 号
- 手机号码
- 家庭住址
- 个人签名

**隐私控制**：

- 每项信息可单独设置"公开/私密"

#### 3.3 点赞管理

**点赞规则**：

- 每日点赞上限：**10 次**
- 不可给自己点赞
- 点赞记录可查看（最近 30 天）

**查看他人信息**：

- 在论坛点击用户头像可查看其公开信息
- 可点赞他人（计入每日上限）
- 可关注用户（可选功能）

***

## 🔗 系统联动

### 与聊天系统联动

1. **帖子转发**：
   - 论坛/热事/美食帖子可转发至私聊或群聊
   - 转发时显示帖子预览（标题 + 部分内容）
   - 点击预览可跳转至原帖子
2. **通知推送**：
   - 帖子被点赞/评论时推送通知
   - 通知可跳转至对应帖子
3. **用户资料同步**：
   - 社区系统与聊天系统共用用户信息
   - 头像、昵称保持一致

### 与其他系统适配

- **资源系统**：可分享学习资源链接
- **管理系统**：后台审核违规内容

## 🎨 UI/UX 设计要求

### 平板横屏适配

**布局建议**：

- **三栏布局**（推荐）：
  - 左侧：导航栏（论坛 / 排行榜/我的）
  - 中间：内容列表
  - 右侧：详情/评论（可选收起）
- **两栏布局**（备选）：
  - 左侧：导航 + 列表
  - 右侧：详情弹窗

**交互设计**：

- 按钮尺寸 ≥ 44px（适合触摸操作）
- 列表项高度 ≥ 56px
- 模态框占屏幕 60%-80% 宽度
- 支持滑动刷新、下拉加载

### 自定义模态框

**禁止使用**：

```javascript
// ❌ 禁止使用
alert('提示');
confirm('确定吗？');
prompt('请输入');
```

**必须使用自定义模态框**：

```javascript
// ✅ 使用自定义组件
<CustomModal title="提示" :visible="showModal">
  <p>确定要删除此帖子吗？</p>
  <div class="modal-actions">
    <button @click="cancel">取消</button>
    <button @click="confirm" class="danger">删除</button>
  </div>
</CustomModal>
```

## 🛡️ 安全与规范

### 内容审核

- **敏感词过滤**：发帖时自动检测并拦截
- **举报机制**：用户可举报违规内容
- **管理员审核**：被举报内容进入审核队列

### 权限控制

| 操作    | 普通用户 | 管理员 |
| ----- | ---- | --- |
| 发帖    | ✅    | ✅   |
| 删己方帖子 | ✅    | ✅   |
| 删他人帖子 | ❌    | ✅   |
| 封禁用户  | ❌    | ✅   |
| 查看举报  | ❌    | ✅   |

### 数据限制

- 单帖文字上限：2000 字
- 每日发帖上限：20 帖
- 图片上传：最多 9 张/帖
- 视频上传：最大 50MB/个

## 📊 数据库设计（建议）

### 用户表 (users)

```sql
- id: 用户 ID
- username: 用户名
- avatar: 头像 URL
- gender: 性别（男/女）
- wechat: 微信号
- qq: QQ 号
- phone: 手机号
- address: 住址
- signature: 个人签名
- privacy_settings: 隐私设置 JSON
```

### 帖子表 (posts)

```sql
- id: 帖子 ID
- user_id: 发布者 ID
- type: 类型（论坛/美食/热事）
- title: 标题
- content: 内容
- anonymous: 是否匿名
- visible_groups: 可见分组 JSON
- hidden_groups: 不可见分组 JSON
- created_at: 发布时间
- updated_at: 更新时间
```

### 点赞表 (likes)

```sql
- id: 记录 ID
- user_id: 点赞者 ID
- target_type: 目标类型（post/comment/user）
- target_id: 目标 ID
- created_at: 点赞时间
- UNIQUE(user_id, target_type, target_id): 防重复点赞
```

### 评论表 (comments)

```sql
- id: 评论 ID
- post_id: 帖子 ID
- user_id: 评论者 ID
- parent_id: 父评论 ID（回复用）
- content: 评论内容
- created_at: 评论时间
```

### 点赞计数缓存表 (like\_counts)

```sql
- target_type: 目标类型
- target_id: 目标 ID
- count: 点赞数
- last_updated: 最后更新时间
```

## 🚀 开发建议

### 前端技术栈

- **框架**：Vue 2.7 + Vite 5
- **状态管理**：Vuex 3.x
- **样式**：Sass + CSS 自定义属性
- **兼容性**：Chromium 78+（禁用 ES2020+）

### 后端技术栈

- **框架**：Express.js
- **数据库**：SQLite (better-sqlite3)
- **认证**：JWT
- **实时通信**：WebSocket (ws)

### 性能优化

1. **列表虚拟滚动**：长列表使用虚拟滚动
2. **图片懒加载**：进入视口再加载
3. **分页加载**：每页 20 条，下拉加载更多
4. **点赞防抖**：快速连续点赞只发送最后一次
5. **缓存策略**：帖子详情缓存 5 分钟
