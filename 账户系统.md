# 账户系统

## 系统概述

账户系统是 ClassNet 的统一身份认证中心，采用单点登录（SSO）机制，实现"一点登录，处处通行"的用户体验。

## 核心功能

### 1. 统一认证机制
- **单点登录（SSO）**: 用户只需登录一次即可访问所有子系统
- **持久化会话**: 登录状态持久保存，除非用户手动登出，否则无需重复登录
- **自动重定向**: 未登录用户访问其他应用时，自动重定向至登录页

### 2. 用户注册

#### 输入信息
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 自定义网名 | String | 是 | 用户自定义的显示名称 |
| 真实姓名 | String | 是 | 用于关联预录入信息 |
| 账号密码 | String | 是 | 用户登录密码（需加密存储） |

#### 注册流程
```
1. 用户填写注册信息（网名、真实姓名、密码）
   ↓
2. 系统验证真实姓名是否在预录入名单中
   ↓
3. 自动关联预录入 ID 和性别信息
   ↓
4. 创建用户账户
   ↓
5. 自动登录并跳转至主页
```

#### 预录入机制
- 系统预先录入用户真实姓名、ID、性别等基础信息
- 注册时通过真实姓名自动匹配预录入数据
- 确保用户信息的准确性和一致性

### 3. 用户登录

#### 支持登录方式
用户可使用以下任一标识进行登录：
- 真实姓名
- 用户 ID
- 网络名称（自定义网名）

#### 登录流程
```
1. 用户输入登录标识（真实姓名/ID/网名）
   ↓
2. 用户输入密码
   ↓
3. 系统验证凭据
   ↓
4. 验证通过，创建会话（Session/Token）
   ↓
5. 跳转至用户主页（桌面）
```

### 4. 会话管理

#### 会话保持
- 使用 Cookie 或 localStorage 存储会话标识
- 会话长期有效，除非用户主动登出
- 支持多标签页共享登录状态

#### 登出机制
- 用户手动登出时清除会话
- 登出后跳转至登录页
- 清除所有本地存储的敏感信息

## 技术要点

### 1. 认证方式
- **推荐方案**: JWT (JSON Web Token) 或 Session-Cookie
- **密码加密**: 使用 bcrypt 或类似算法进行密码哈希
- **传输安全**: 使用 HTTPS 传输认证信息

### 2. 权限验证中间件
```javascript
// 伪代码示例
function authMiddleware(req, res, next) {
  if (!req.session || !req.session.userId) {
    return res.redirect('/login');
  }
  next();
}
```

### 3. 用户数据结构
```javascript
{
  id: "用户唯一标识",
  username: "自定义网名",
  realName: "真实姓名",
  preRecordedId: "预录入 ID",
  gender: "性别",
  passwordHash: "密码哈希值",
  createdAt: "注册时间",
  status: "账户状态（正常/禁用）"
}
```

## 安全注意事项

1. **密码安全**
   - 密码必须加密存储，禁止明文保存
   - 使用强密码策略（长度、复杂度要求）

2. **会话安全**
   - 设置合理的会话过期时间
   - 使用 HttpOnly Cookie 防止 XSS 攻击
   - 实施 CSRF 保护机制

3. **输入验证**
   - 对所有输入进行验证和过滤
   - 防止 SQL 注入和 XSS 攻击

## 与其他系统的关系

| 关联系统 | 关联方式 | 说明 |
|---------|---------|------|
| 所有子系统 | 认证依赖 | 所有系统依赖账户系统进行身份验证 |
| 管理系统 | 数据提供 | 为管理系统提供用户数据支持 |
| 聊天系统 | 用户信息 | 提供聊天用户的身份信息 |
